### Undersøg: GitLab Continuous Integration & Deployment
### Hvad betyder CI og hvordan kan man bruge versionsstyrring til dette?
CI står for Continuous Integration og betyder at når man pusher til repo vil der blive kørt en række scripts/checks for at fx. sikre at standarder er opholdt eller tjekke om der er fejl i koden

### Undersøg GitHub / Gitlab
### Hvad er forskellene?
Den største forksel mellem GitHub og Gitlab er at GitHub har primært været fokuseret på udelukkende code hosting, hvor Gitlab er mere fokuseret på at være en Dev-Ops platform med CI/CD indbygget.

### Er der fordele og ulemper ved at bruge den ene frem for den anden?
GitHub har generalt et hurtiger / mere bruger-venlidt og streamlined interface dov med mindre support for et mere avanceret Dev-Ops environment.
